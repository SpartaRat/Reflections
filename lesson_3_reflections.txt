Quando você gostaria de criar um repositório remoto em vez de manter todo seu trabalho local?
	Quando colaborando com outros desenvolvedores fica mais fácil compartilhar updates entre toda a equipe.
Por que você prefere efetuar o pull de alterações manualmente do que deixar o Git atualizar-se automaticamente com o seu repositorio remoto?
	Porque assim eu posso visualizar primeiramente as alterações que foram feitas no GitHub e depois decidir se essas alterações não causarão um bug no meu sistema, depois de confimar que essas novas alterações não causarão danos então atualizo meu repositório local. Se esse processo acontecesse automaticamente poderia ocorrer de vez receber um versão bugada enquanto trabalha em um outro recurso.
Descreva a diferença entre bifurcações, clones e branches. Quando você usaria um e não os outros?
	Bifurcações: São para clonar um repositório Do GitHub em outro.
	Clones: São para fazer uma cópia de um repositório do GitHub no diretório local
	Branches: São usados em todos os repositórios para ramificar o branch master.
Qual é o benefício de ter uma cópia do último estado conhecido do remoto armazenado localmente?
	É saber quando as suas alterações não estão salvas na nuvem, além de ser um ponto em comum entre os commits criados localmente e os que foram criados por outros desenvolvedores e colocados no GitHub
